以下の手順でテストを実行してください：

1. 現在のファイルに対するテストファイルを探す
   - ファイル名が`foo.ts`の場合、`foo.test.ts`、`foo.spec.ts`、`__tests__/foo.ts`などを探す
   - テストファイルが見つからない場合は、標準的な場所（同じディレクトリまたは`__tests__`ディレクトリ）を確認

2. テストが存在する場合は実行する
   - `npm test`または適切なテストコマンドを使用
   - テスト結果を詳細に報告（成功/失敗の数、エラーメッセージなど）

3. テストが存在しない場合は、適切なテストファイルを作成する
   - ファイルの内容を分析し、主要な関数やコンポーネントを特定
   - Jest/Vitestを使用した包括的なテストを作成
   - 以下をカバーするテストを含める：
     - 正常系のテスト
     - 異常系のテスト（エラーハンドリング）
     - エッジケース
     - 型の検証（TypeScriptの場合）

4. テスト結果を報告する
   - テストの成功/失敗の概要
   - 失敗したテストがある場合は、原因と修正方法を提案
   - カバレッジ情報があれば表示

## 注意事項

- Next.jsプロジェクトなので、React Testingライブラリを使用する場合があります
- Server Actionsやクライアントコンポーネントのテスト方法に注意
- 環境変数が必要な場合は、テスト用のモックを使用
- 非同期関数のテストには`async/await`を適切に使用
